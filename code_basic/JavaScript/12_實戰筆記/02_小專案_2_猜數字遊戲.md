# 小專案 2：猜數字遊戲

**核心概念**: 亂數生成、條件判斷、迴圈、使用者輸入、DOM 操作

猜數字遊戲是一個互動性強的小專案，非常適合練習 JavaScript 的基本邏輯控制和使用者介面互動。

## 專案功能點

1.  **生成秘密數字**：電腦在一個指定範圍內（例如 1 到 100）隨機生成一個秘密數字。
2.  **使用者輸入**：提供一個輸入框讓使用者猜測數字。
3.  **遊戲回饋**：根據使用者的猜測，給出提示：「猜大了」、「猜小了」、「恭喜你，猜對了！」
4.  **猜測次數限制**：可選，限制使用者猜測的次數。
5.  **遊戲重置**：提供一個按鈕讓使用者可以重新開始遊戲。
6.  **顯示歷史猜測**：記錄使用者每次猜測的數字。

## 主要技術點

### 1. HTML 結構 (基本骨架)

*   `input` 元素用於接收猜測數字。
*   `button` 元素用於提交猜測。
*   `p` 或 `div` 元素用於顯示遊戲回饋訊息。
*   `button` 元素用於重新開始。
*   `ul` 或 `div` 用於顯示歷史猜測。

```html
<!-- 範例 HTML 骨架 -->
<div class="game-container">
    <h1>猜數字遊戲</h1>
    <p>猜一個 1 到 100 之間的數字。</p>
    <input type="number" id="guessInput" min="1" max="100">
    <button id="submitGuess">提交猜測</button>
    <p id="feedback" class="message"></p>
    <p>你的猜測歷史: <span id="guesses"></span></p>
    <button id="resetGame">重新開始</button>
</div>
```

### 2. JavaScript 邏輯

#### (1) 初始化遊戲狀態

*   `secretNumber`: 透過 `Math.random()` 和 `Math.floor()` 生成 1 到 100 的亂數。
*   `guessesMade`: 記錄已猜測次數，初始化為 0。
*   `previousGuesses`: 陣列，儲存每次猜測的數字。
*   DOM 元素引用：獲取所有必要的按鈕、輸入框和顯示區域。

#### (2) 生成亂數

```javascript
function generateRandomNumber(min, max) {
    return Math.floor(Math.random() * (max - min + 1)) + min;
}
let secretNumber = generateRandomNumber(1, 100);
```

#### (3) 處理使用者猜測 (事件監聽)

*   監聽提交按鈕的 `click` 事件，或輸入框的 `keydown` 事件 (當按下 Enter 時)。
*   在事件處理函式中：
    *   獲取使用者輸入的數字 (`input.value`)，並確保轉換為數字 (`parseInt()` 或 `Number()`)。
    *   檢查輸入的有效性（是否在範圍內，是否為數字）。
    *   更新 `guessesMade` 和 `previousGuesses`。
    *   **條件判斷**：
        *   如果猜對了：顯示「恭喜！」訊息，禁用輸入和提交按鈕，顯示重置按鈕。
        *   如果猜錯了：顯示「猜大了」或「猜小了」訊息。
        *   檢查是否達到猜測次數上限（如果有設定）。

#### (4) 顯示遊戲回饋與歷史

*   更新 `feedback` 元素的 `textContent`。
*   將每次猜測的數字添加到 `guesses` 元素的 `textContent` 或 `innerHTML` 中。

#### (5) 重新開始遊戲

*   監聽重置按鈕的 `click` 事件。
*   重置所有遊戲狀態變數（`secretNumber`, `guessesMade`, `previousGuesses`）。
*   清空輸入框和所有回饋訊息。
*   啟用輸入和提交按鈕，隱藏重置按鈕。

## 3. CSS 樣式 (簡述)

*   美化遊戲介面。
*   為不同的回饋訊息提供不同顏色或樣式。

## 4. 進階考量

*   **輸入驗證**：更嚴格的輸入檢查。
*   **音效**：猜對/猜錯時播放音效。
*   **難度選擇**：允許使用者選擇數字範圍或猜測次數。
*   **響應式設計**：確保在不同大小的螢幕上都能良好顯示。
