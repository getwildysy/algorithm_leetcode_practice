# 小專案 1：待辦清單 (Todo List)

**核心概念**: DOM 操作、事件處理、資料儲存 (LocalStorage)

一個待辦清單是一個經典的前端專案，它涵蓋了許多基本的 JavaScript 知識點。

## 專案功能點

1.  **新增待辦事項**：使用者可以輸入文字，點擊按鈕或按下 Enter 鍵新增事項。
2.  **顯示待辦事項**：將新增的事項顯示在清單中。
3.  **標記完成**：點擊事項或特定的按鈕，可以將事項標記為完成（例如加上刪除線，或移動到另一個清單）。
4.  **刪除待辦事項**：點擊刪除按鈕，移除該事項。
5.  **篩選待辦事項**：顯示「所有」、「未完成」、「已完成」事項。
6.  **資料持久化**：將待辦事項儲存到 `localStorage`，即使關閉瀏覽器，資料也不會丟失。

## 主要技術點

### 1. HTML 結構 (基本骨架)

*   `input` 元素用於輸入新事項。
*   `button` 元素用於新增。
*   `ul` 或 `div` 用於顯示待辦事項列表。
*   每個待辦事項通常由 `li` 包含文字、完成按鈕和刪除按鈕。
*   篩選器可以是 `select` 元素或一系列按鈕。

```html
<!-- 範例 HTML 骨架 -->
<div class="todo-container">
    <h1>我的待辦清單</h1>
    <form id="todo-form">
        <input type="text" id="todo-input" placeholder="新增待辦事項...">
        <button type="submit">新增</button>
    </form>
    <div class="filters">
        <button data-filter="all">所有</button>
        <button data-filter="pending">未完成</button>
        <button data-filter="completed">已完成</button>
    </div>
    <ul id="todo-list">
        <!-- 待辦事項會動態新增到這裡 -->
    </ul>
</div>
```

### 2. JavaScript 邏輯

#### (1) 元素查詢與事件監聽

*   使用 `document.querySelector` 或 `getElementById` 獲取 DOM 元素。
*   使用 `addEventListener` 監聽表單提交、按鈕點擊等事件。

#### (2) 處理新增事項

*   獲取 `input` 的值。
*   建立新的 `li` 元素，並設定其 `textContent`。
*   新增完成按鈕和刪除按鈕。
*   將新建立的 `li` 元素 `appendChild` 到 `ul` 列表中。
*   清空 `input`。

#### (3) 處理完成與刪除

*   為每個待辦事項的完成按鈕和刪除按鈕添加事件監聽器。
*   **事件委派**：考慮在父元素 (例如 `ul`) 上監聽點擊事件，然後根據 `event.target` 判斷是點擊了完成還是刪除按鈕，這樣可以減少監聽器數量，特別是當待辦事項數量很多時。
*   **完成**：透過改變 class 來切換樣式 (例如 `line-through`)。
*   **刪除**：使用 `element.remove()` 移除 DOM 元素。

#### (4) 資料持久化

*   使用 `localStorage.setItem(key, JSON.stringify(data))` 儲存待辦事項陣列。
*   使用 `localStorage.getItem(key)` 讀取資料，並 `JSON.parse()` 轉換回陣列。
*   在應用程式啟動時，從 `localStorage` 載入資料並渲染到頁面。
*   每次新增、修改、刪除事項時，都要更新 `localStorage` 中的資料。

#### (5) 篩選功能

*   監聽篩選按鈕的點擊事件。
*   根據選定的篩選條件，遍歷待辦事項列表，顯示或隱藏相應的項目。

## 3. CSS 樣式 (簡述)

*   使用 CSS 美化界面。
*   定義 `.completed` class 來為完成的事項添加樣式。

## 4. 進階考量

*   **唯一 ID**：為每個待辦事項生成唯一 ID，便於管理和刪除。
*   **動畫效果**：使用 CSS 動畫或 JavaScript 動畫庫提升使用者體驗。
*   **可訪問性 (Accessibility)**：考慮使用 ARIA 屬性。
*   **前端框架**：在實際專案中，通常會使用 React, Vue, Angular 等框架來簡化 DOM 操作和狀態管理。
