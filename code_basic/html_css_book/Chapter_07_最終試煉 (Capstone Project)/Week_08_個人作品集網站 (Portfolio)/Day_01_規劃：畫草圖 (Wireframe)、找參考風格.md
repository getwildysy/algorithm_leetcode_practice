# Day 1: 規劃：畫草圖 (Wireframe)、找參考風格

這週是你的 Final Project，我們的目標是做出一個完整的 **個人作品集網站 (Portfolio)**。
在寫任何一行程式碼之前，最重要的一步是「規劃」。

## 1. 網站地圖 (Sitemap)

一個標準的單頁式 (Single Page) 作品集通常包含：
1.  **Hero Section**: 大頭照、你的名字、一句話自我介紹 (Slogan)。
2.  **About Me**: 詳細背景、技能表 (Skill Set)。
3.  **Projects**: 你的作品展示 (卡片式排版)。
4.  **Contact**: 聯絡表單或社群連結。

## 2. 線框圖 (Wireframe)

拿起紙筆，畫出這四個區塊的大致排版。
*   Hero 要置中還是左右分欄？
*   Projects 要一排三個還是一排兩個？
*   手機版選單要用漢堡嗎？

> **Tip**: 其實不用畫太細，只要確定「方塊在哪裡」就好。

## 3. 尋找靈感 (Mood Board)

去 Pinterest 或 Dribbble 搜尋 "Developer Portfolio"。
決定你的色系與風格：
*   **極簡風 (Minimalist)**: 大量留白、黑白灰。
*   **科技風 (Tech)**: 黑色背景、霓虹綠/藍色點綴。
*   **活潑風 (Playful)**: 鮮豔色彩、大圓角、有趣的字體。

## 4. 今日作業
不要打開 VS Code。
請在紙上畫出你的網站草圖，並選定 3 個主要顏色 (主色、副色、背景色)。
