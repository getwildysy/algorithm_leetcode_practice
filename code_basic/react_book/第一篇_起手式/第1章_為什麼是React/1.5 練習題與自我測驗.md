# 1.5 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: 關於 React 的 Virtual DOM，下列敘述何者正確？
1. Virtual DOM 總是比真實 DOM 操作慢，但為了開發方便而存在。
2. Virtual DOM 是真實 DOM 的一份記憶體中複製品，React 用它來計算最小更新範圍。
3. 每次 State 改變，React 都不會重新渲染 Virtual DOM，而是直接修改真實 DOM。
4. Virtual DOM 是瀏覽器原生的功能。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：Virtual DOM 透過 Reconciliation 演算法比對前後差異，只更新變動的部分，這是 React 效能的關鍵。
</details>

---

### Q2: 下列哪個不是 React 宣告式 (Declarative) UI 的特性？
1. 我們只需描述 UI 在某個狀態下「長什麼樣子」。
2. 我們需要手動操作 DOM (例如 `append()`, `remove()`) 來更新畫面。
3. State 改變時，UI 會自動響應更新。
4. 程式碼通常比指令式更簡潔易讀。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：手動操作 DOM 是指令式 (Imperative) 的特徵，React 負責處理 DOM 操作的細節。
</details>

### Q3: 為什麼現代開發推薦使用 Vite 取代 Create React App？
1. 因為 Vite 是由 Facebook 官方維護的。
2. 因為 Vite 使用 Go 語言編寫的 esbuild 進行預構建，開發伺服器啟動極快。
3. 因為 Vite 不支援 TypeScript，學習門檻較低。
4. 因為 Webpack 已經停止運作了。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：Vite 利用 Native ESM 和 esbuild 大幅提升了開發環境的效能。
</details>

## 程式碼挑戰 (Coding Challenges)

請打開我們建立的 `hello-react` 專案，嘗試完成以下任務：

### 挑戰 1：動態時間顯示
**目標**：在畫面上顯示一個會每秒更新的時鐘。

**提示**：
1. 你需要 `useState` 來儲存時間。
2. 你需要 `useEffect` 來設定 `setInterval`。
3. 記得 `new Date().toLocaleTimeString()` 可以取得目前時間字串。

### 挑戰 2：切換開關 (Toggle Button)
**目標**：製作一個按鈕，點擊後文字在 "開 (ON)" 與 "關 (OFF)" 之間切換，且背景顏色隨之改變（例如開是綠色，關是紅色）。

**提示**：
1. 使用一個 boolean state，例如 `const [isOn, setIsOn] = useState(false)`。
2. 根據 `isOn` 的值動態設定 `style` 或 `className`。

### 挑戰 3：清單過濾器
**目標**：顯示一個水果清單，並提供一個輸入框，輸入文字時只顯示包含該文字的水果。

**提示**：
1. 準備一個陣列 `const fruits = ['Apple', 'Banana', 'Cherry', 'Date', 'Elderberry']`。
2. 使用一個 state 儲存搜尋關鍵字。
3. 使用 `filter()` 方法來產生要顯示的清單。
