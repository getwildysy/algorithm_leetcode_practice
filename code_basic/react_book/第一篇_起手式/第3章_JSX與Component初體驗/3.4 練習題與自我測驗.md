# 3.4 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: [JSX] 下列哪個是合法的 JSX 語法？
1. `<div class="box">Content</div>`
2. `<div>{ if (ok) return 'Yes' }</div>`
3. `<img src="logo.png">`
4. `<button onClick={() => alert('Hi')}>Click</button>`

<details>
<summary>點擊查看答案</summary>
**答案：4**
<br>
**解析**：
1. `class` 應為 `className`。
2. JSX 大括號內不能放 if 陳述句 (Statement)，只能放表達式 (Expression)。
3. `img` 必須閉合 (`<img />`)。
</details>

### Q2: [Props] 關於 Props 的描述，何者**錯誤**？
1. Props 是唯讀的 (Read-only)，子元件不能直接修改它。
2. Props 可以傳遞字串、物件、甚至是函式。
3. `children` 是一個特殊的 Prop，代表標籤中間的內容。
4. 子元件可以直接修改 Props 傳進來的物件屬性，父元件會自動更新。

<details>
<summary>點擊查看答案</summary>
**答案：4**
<br>
**解析**：這違反了單向資料流原則，且 React 的 Props 應該被視為不可變的。雖然技術上修改物件屬性可能不會報錯 (Reference 無變)，但這是嚴重的 Anti-Pattern。
</details>

---

### Q3: [Component] 在 Atomic Design 中，`SearchBar` (包含 Input 和 Button) 通常被歸類為哪一層級？
1. Atom (原子)
2. Molecule (分子)
3. Organism (組織)
4. Page (頁面)

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：Atom 是不可拆分的 (如單獨的 Input)，Organism 是複雜區塊 (如 Header)，Molecule 則是簡單的功能組合。
</details>

## 程式碼挑戰 (Coding Challenges)

請在接下來的實戰演練專案中嘗試這些挑戰。

### 挑戰 1：條件渲染 UserBadge
**目標**：建立一個 `UserBadge` 元件。
- 接收 `name` (string) 和 `isPro` (boolean) 兩個 props。
- 顯示名字。
- 如果 `isPro` 為 true，在名字旁邊顯示一個金色的 "PRO" 標籤。

```jsx
// 範例使用
<UserBadge name="Alice" isPro={true} />
```

### 挑戰 2：實作 ImageCard
**目標**：利用 `children` prop 建立一個通用的卡片容器。
- 建立 `Card` 元件，負責外框陰影和圓角。
- 建立 `CardImage` 元件，負責圖片滿版顯示。
- 建立 `CardContent` 元件，負責內距 (Padding)。
- 像積木一樣組合它們。

```jsx
<Card>
  <CardImage src="photo.jpg" />
  <CardContent>
    <h3>Beautiful View</h3>
  </CardContent>
</Card>
```

### 挑戰 3：顏色選擇器 (Props 傳遞函式)
**目標**：練習子元件通知父元件。
- 父元件 `App` 有一個背景色 state。
- 子元件 `ColorPicker` 顯示三個顏色按鈕 (紅/藍/綠)。
- 點擊按鈕時，父元件的背景色要跟著變。

**提示**：
父元件傳遞一個 `onColorChange` 函式給子元件。
