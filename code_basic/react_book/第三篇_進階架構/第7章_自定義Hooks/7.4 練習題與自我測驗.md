# 7.4 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: [Rule] 關於 Custom Hook 的命名，下列何者正確？
1. 必須以 `hook` 開頭，例如 `hookUser`。
2. 必須以 `use` 開頭，例如 `useUser`。
3. 隨便命名都可以，只要能夠跑就好。
4. 必須全大寫，例如 `USE_USER`。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：這是 React 官方約定，也是 ESLint 插件識別 Hook 的依據。
</details>

---

### Q2: [Logic] 兩個元件同時使用了 `useCounter` 這個 Hook，請問它們的狀態是共用的嗎？
1. 是，所有使用該 Hook 的元件共享同一個 count 變數。
2. 否，每次呼叫 Hook 都會產生獨立的 State，互不影響。
3. 只有在同一個父元件下才會共用。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：Custom Hook 只是邏輯的複用，不是狀態的複用。如果需要共享狀態，需要使用 Context API 或 Redux。
</details>

## 程式碼挑戰 (Coding Challenges)

### 挑戰 1：useToggle
**目標**：封裝一個簡單的開關 Hook。
- 輸入：初始布林值 (預設 false)。
- 輸出：`[isToggled, toggle, setToggle]`。
- `toggle` 函式不需要參數，直接反轉狀態。

### 挑戰 2：useDocumentTitle
**目標**：同步網頁標題。
- 當傳入的 title 改變時，自動更新 `document.title`。
- 當元件卸載時，嘗試還原原本的標題 (選做)。

### 挑戰 3：useFetch (含 Cache)
**目標**：改進範例中的 useFetch。
- 加入一個簡單的記憶體 Cache (全域物件)。
- 當 URL 請求過一次後，第二次請求直接回傳 Cache 中的資料，不發送網路請求。
