# 8.4 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: [Context] 關於 Context API，下列何者正確？
1. 它是用來取代 Redux 的，效能比 Redux 好。
2. 只要 Context Value 變動，所有使用 `useContext` 的子元件都會重新渲染。
3. Context 只能傳遞字串，不能傳遞物件或函式。
4. 一個 React App 只能有一個 Context。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：這是 Context 最大的效能隱憂，因此不適合存放高頻變動的狀態。
</details>

---

### Q2: [Zustand] 為什麼 Zustand 比 Context 更適合做複雜狀態管理？
1. 因為它的名字比較帥。
2. 因為它支援 Selector 機制，元件可以只訂閱 Store 中的一部分資料，避免不必要的 Re-render。
3. 因為它是由 Facebook 官方開發的。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：Zustand 的 Selector 機制解決了 Context "Over-rendering" 的問題。
</details>

## 程式碼挑戰 (Coding Challenges)

### 挑戰 1：Context 實作登入系統
**目標**：不使用任何套件，用 Context 實作 Auth 功能。
- 建立 `AuthContext`。
- 提供 `user` 狀態 (null 代表沒登入)。
- 提供 `login(username)` 和 `logout()` 方法。
- 在 `Header` 顯示 "Welcome, [username]" 或 "Login" 按鈕。

### 挑戰 2：Zustand 實作計數器
**目標**：練習 Zustand 基本語法。
- 建立一個 Store，包含 `count`、`increment`、`decrement`、`reset`。
- 建立兩個元件：
  - `Display`: 顯示目前數字。
  - `Controls`: 包含三個按鈕。
- 觀察：點擊按鈕時，只有 Display 會 Render 嗎？(可以使用 `console.log` 觀察)。

### 挑戰 3：用 Context 解決 Prop Drilling
**情境**：
App -> Dashboard -> Widget -> Settings -> **ColorPicker**
App 有一個 `themeColor` 狀態，想要傳給 **ColorPicker** 使用。
請重構這個傳遞鏈，改用 Context 直接傳遞。
