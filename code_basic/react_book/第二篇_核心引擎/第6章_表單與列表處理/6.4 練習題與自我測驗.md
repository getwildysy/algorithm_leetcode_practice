# 6.4 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: [Key] 為什麼不建議使用陣列索引 (Index) 作為 Key？
1. 因為 Index 不是數字。
2. 因為 Index 太短了。
3. 因為當列表順序改變或刪減時，Index 會對應到錯誤的元件實例，導致狀態錯亂。
4. 因為 React 官方討厭 Index。

<details>
<summary>點擊查看答案</summary>
**答案：3**
<br>
**解析**：Reconciliation 依賴 Key 來識別身分。如果刪除了第 0 項，原本的第 1 項 Index 變成了 0，React 會以為它是同一個元件，導致狀態移花接木。
</details>

---

### Q2: [Form] 什麼是受控元件 (Controlled Component)？
1. 由瀏覽器控制的元件。
2. 由 React State 作為唯一資料來源 (Source of Truth) 的元件。
3. 只能讀不能寫的元件。
4. 使用 `ref` 操作的元件。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：透過 `value={state}` 和 `onChange={setState}` 將 Inputs 的狀態交給 React 接管。
</details>

```jsx
const [skills, setSkills] = useState([{ id: 1, name: 'React' }]);

// 用 skills.map 渲染...
```

### 挑戰 2：即時密碼強度檢測
**目標**：
- 一個密碼 Input。
- 下方顯示強度條：
  - < 6 碼：紅色 (弱)
  - > 6 碼且含數字：黃色 (中)
  - > 8 碼且含數字與符號：綠色 (強)
- 嘗試用受控元件實作。

### 挑戰 3：全選/全不選 (Checkbox Group)
**目標**：
- 一個 "Select All" Checkbox。
- 下面有三個 Item Checkbox (A, B, C)。
- 規則：
  1. 點 Select All -> 下面全勾/全取消。
  2. 下面全勾時 -> Select All 自動打勾。
  3. 下面只要有一個沒勾 -> Select All 自動取消。
