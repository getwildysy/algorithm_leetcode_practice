在學習 React Router 之前，我們必須先理解現代前端開發的核心架構 —— **Single Page Application (SPA)**。

## 1. 傳統多頁應用 (MPA) vs. SPA

### 傳統 MPA (Multiple Page Application)
每次點擊連結 (例如 `about.html`)，瀏覽器會：
1. 向伺服器發送 Request。
2. 伺服器回傳**全新的 HTML**。
3. 瀏覽器**重新刷新**整個頁面 (白屏閃爍)。
4. 下載該頁面所需的 CSS/JS。

### 現代 SPA (Single Page Application)
整個網站只有一個 HTML (index.html)。每次點擊連結：
1. **JavaScript 攔截**點擊事件 (preventDefault)。
2. 只去**更換**頁面中需要變動的元件 (例如把 `<Home>` 換成 `<About>`)。
3. 網址變了 (`/about`)，但瀏覽器**沒有刷新**。

## 2. 為什麼需要「路由 (Router)」？

既然只有一個 HTML，那我們怎麼讓使用者覺得他在逛不同的頁面？而且當使用者按「上一頁」時，要怎麼回到上個狀態？

這就是 Router 的工作：
1. **監聽 URL 變化**：當網址變成 `/about` 時。
2. **渲染對應元件**：告訴 React 該渲染 `<About />`。
3. **管理歷史紀錄 (History)**：讓瀏覽器的上一頁/下一頁按鈕能正常運作。

React Router 是目前 React 生態系中最標準的解決方案 (React Router v6)。

## 小結

- **SPA** 體驗像 App，流暢無白屏。
- **Router** 是 SPA 的導航員，負責把 URL 對應到正確的 Component。
