# 10.2 React Router 基礎

React Router v6 帶來了很大的變革，語法變得更簡潔直觀。

## 1. 安裝與設定

`npm install react-router-dom`

### BrowserRouter
在 `main.jsx` (或 `index.js`) 中，我們必須把 `App` 包在 `<BrowserRouter>` 裡面，這樣整個 App 才能啟用路由功能。

```jsx
import { BrowserRouter } from 'react-router-dom';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);
```

## 2. 定義路由 (Routes & Route)

在 `App.jsx` 中，我們使用 `<Routes>` 和 `<Route>` 來定義網址與元件的對應關係。

```jsx
import { Routes, Route } from 'react-router-dom';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/about" element={<About />} />
      {/* 404 頁面 (Catch all) */}
      <Route path="*" element={<NotFound />} />
    </Routes>
  );
}
```

## 3. 頁面跳轉 (Link vs a)

在 SPA 中，**絕對不可以使用 `<a>` 標籤**！因為那會導致瀏覽器重新刷新頁面，破壞 SPA 的體驗 (State 會清空)。

請使用 `<Link>` 或 `<NavLink>`：

```jsx
import { Link, NavLink } from 'react-router-dom';

// 普通連結
<Link to="/about">Go to About</Link>

// 帶有 active 狀態的連結 (適合導航列)
<NavLink 
  to="/about" 
  className={({ isActive }) => isActive ? "active-link" : ""}
>
  About
</NavLink>
```

## 小結

- **BrowserRouter**: 路由的容器。
- **Routes / Route**: 定義 URL -> Component 的對照表。
- **Link**: SPA 專用的超連結 (取代 `<a>`)。
