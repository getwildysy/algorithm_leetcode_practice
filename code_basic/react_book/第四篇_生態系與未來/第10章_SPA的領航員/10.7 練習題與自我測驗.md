# 10.7 練習題與自我測驗

## 觀念測驗 (Multiple Choice)

### Q1: [Link] 在 React Router 中，為什麼要使用 `<Link>` 而不是 `<a>` 標籤？
1. 因為 `<a>` 標籤不支援 CSS 樣式。
2. 因為 `<a>` 標籤會觸發瀏覽器的完整頁面刷新 (Full Page Reload)，導致 React 應用重新初始化，State 遺失。
3. 因為 `<Link>` 會自動加密網址。

<details>
<summary>點擊查看答案</summary>
**答案：2**
<br>
**解析**：SPA 的核心體驗就是「無刷新切換」，這需要依賴 JavaScript 攔截點擊事件 (preventDefault) 並操作 History API，這正是 `<Link>` 做的事。
</details>

### Q2: [Dynamic] 如果定義了路由 `<Route path="/users/:id" ... />`，當網址為 `/users/9527` 時，要如何取得 `9527` 這個值？
1. `useParams().id`
2. `useSearchParams().get('id')`
3. `props.match.params.id`
4. `window.location.id`

<details>
<summary>點擊查看答案</summary>
**答案：1**
<br>
**解析**：React Router v6 使用 `useParams` Hook 來存取路徑參數。
</details>

## 程式碼挑戰 (Coding Challenges)

### 挑戰 1：Active Link Style
**目標**：製作一個導航列 (`Navbar`)。
- 包含三個連結：Home, About, Contact。
- 當使用者在 "About" 頁面時，"About" 的連結要變成紅色且加粗。
- 提示：使用 `NavLink` 和它的 `isActive` 屬性。

### 挑戰 2：Route Guard (路由守衛)
**目標**：保護 `/dashboard` 頁面。
- 建立一個 `PrivateRoute` 元件 (或 Wrapper)。
- 檢查 `isLoggedIn` 狀態。
- 如果沒登入，自動將使用者引導 (`Navigate`) 回 `/login` 頁面。
- 如果有登入，顯示目標內容 (`children`)。

### 挑戰 3：麵包屑導航 (Breadcrumbs)
**目標**：根據目前的 URL 路徑，自動產出麵包屑。
- 例如網址 `/products/electronics/iphone`。
- 顯示： `Home > Products > Electronics > iPhone`。
- 每個層級都要能點擊回到該層級。
- 提示：使用 `useLocation` 取得路徑字串並 split。
